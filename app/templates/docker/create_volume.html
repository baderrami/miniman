Notes:
  • Volumes are the preferred mechanism for persisting data generated by and used by Docker containers.
  • Volumes are completely managed by Docker and are isolated from the core functionality of the host machine.
  • A volume can be mounted into multiple containers simultaneously.
  • When a container is removed, any volume it was using persists.
Volume Drivers:
  • local: Default driver that stores data on the host filesystem.
  • nfs: Allows mounting NFS shares as Docker volumes.
  • tmpfs: Mounts a temporary filesystem in memory (data is lost on host restart).
