Connect a Container to a Network
Select the container to connect to the network.
Select the network to connect the container to.
Notes:
  • Connecting a container to a network allows it to communicate with other containers on the same network.
  • Containers can be connected to multiple networks.
  • The container must be running to connect it to a network.
  • Some network drivers may have restrictions on which containers can be connected.
Network Types:
  • bridge: Default network driver that allows containers on the same host to communicate.
  • host: Removes network isolation between the container and the host.
  • overlay: Connects multiple Docker daemons together for Swarm services.
  • macvlan: Assigns a MAC address to a container, making it appear as a physical device on the network.
  • none: Disables all networking for the container.
