Create a New Network
Enter a name for the network. Use only letters, numbers, dashes, and underscores.
Select the network driver to use. The default is 'bridge'.
Enter a subnet in CIDR notation (e.g., 192.168.0.0/24).
Enter a gateway IP address for the subnet.
Notes:
  • Networks allow containers to communicate with each other and with the outside world.
  • The bridge driver is the default and creates a private network internal to the host.
  • The host driver removes network isolation between the container and the host.
  • The overlay driver creates a network that spans multiple Docker hosts.
  • The macvlan driver assigns a MAC address to a container, making it appear as a physical device on the network.
  • The none driver disables all networking for the container.
Subnet and Gateway:

If you don't specify a subnet and gateway, Docker will automatically assign them. However, if you need specific IP addressing, you can specify them here.
