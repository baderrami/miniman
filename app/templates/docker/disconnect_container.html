Disconnect a Container from a Network
Select the container to disconnect from the network.
Select the network to disconnect the container from.
Notes:
  • Disconnecting a container from a network will prevent it from communicating with other containers on that network.
  • The container will still be able to communicate with containers on other networks it is connected to.
  • You cannot disconnect a container from the default bridge network if it is the only network the container is connected to.
  • Disconnecting a container from a network does not affect its running state.
Warning:

Disconnecting a container from a network may disrupt services that depend on network communication. Make sure you understand the implications before proceeding.
